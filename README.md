# 中職對戰｜線上文字棒球對決

**線上雙人**逐球猜球心理戰棒球遊戲。兩位玩家各自用自己的手機或電腦，透過 4 位數房號配對對戰。Next.js 14 + Tailwind CSS + Upstash Redis，部署在 Vercel。

## 怎麼玩

1. 玩家 A 打開網站 →「建立房間」→ 選局數和球隊 → 拿到 4 位數房號
2. 玩家 B 打開同一個網站 →「加入房間」→ 輸入房號、選球隊 → 比賽自動開始
3. 每一球：守備方配球（S/A/B/C 球種等級＋拖曳落點＋出手時機＋佈陣，同時進攻方可偷偷下盜壘暗號）→ 進攻方即時反應（球影閃現九宮格，把甜蜜點拖到那一格點下去）→ 雙方同時看到結果 → 都按「下一球」繼續
4. 因為兩人各看各的螢幕，配球內容天然保密（打者只看得到守備佈陣），心理戰完全成立

## ⚠️ 部署必看：需要連接一個免費資料庫

線上對戰需要一個地方存「房間狀態」讓兩台裝置同步，我們用 **Upstash Redis**（有免費額度，這個遊戲的用量絕對用不完）。**沒設定資料庫的話，建立房間會失敗。**

### 最簡單的方式：從 Vercel Marketplace 一鍵安裝

1. 先照下面「部署到 Vercel」把專案部署起來（第一次部署會成功，只是遊戲還不能建房）
2. 到 Vercel 你的專案頁 → 上方「**Storage**」分頁 → 找到「**Upstash**」（Serverless Redis）→ 按「**Create**」
3. 照畫面指示建立一個免費的 Redis 資料庫（Region 選 Singapore 或任何離台灣近的）
4. 建立時 Vercel 會問要連接到哪個專案 → 選你這個遊戲專案 → **Connect**
5. 回到專案頁 → 「**Deployments**」分頁 → 最新那筆右邊「⋯」→「**Redeploy**」

完成！環境變數（`KV_REST_API_URL`、`KV_REST_API_TOKEN`）會自動設定好，重新部署後就能建房對戰了。

> 程式同時相容 Upstash 官網直接建立的變數名稱（`UPSTASH_REDIS_REST_URL`、`UPSTASH_REDIS_REST_TOKEN`），兩種方式都可以。

## 部署到 Vercel

1. 把這個資料夾的**內容物**上傳到一個 GitHub repo（注意：是裡面的檔案們，不是 zip、也不是外層資料夾）
2. 到 https://vercel.com/new → Import 該 repo → Framework 自動偵測 Next.js → **Deploy**
3. 完成上面的「連接 Upstash Redis」步驟
4. 拿到 `*.vercel.app` 網址，傳給朋友開戰

之後改任何檔案（例如調整球員數值），只要在 GitHub 上編輯並 Commit，Vercel 會自動重新部署。

## 本機開發

```bash
npm install
npm run dev
```

本機測試線上功能需要 Redis 連線資訊：在專案根目錄建立 `.env.local`：

```
UPSTASH_REDIS_REST_URL=你的網址
UPSTASH_REDIS_REST_TOKEN=你的token
```

（到 upstash.com 免費註冊建一個 Redis 資料庫即可取得。）

## 遊戲機制總覽

- **選擇時限**：投手配球與打者反應各 25 秒。投手超時＝失投紅中直球；打者超時＝自動不揮棒（任一方輪詢即觸發判定，斷線也擋不住比賽前進）
- **投手出手時機**：拖曳配球後進入出手時機條（光標擺動、按下出手）。出手品質微調實際落點；出手極差＝失投紅中
- **打者即時反應（真實揮棒）**：按住畫面＝持棒瞄準（游標是一支球棒，棒身上的球標＝甜蜜點），**放開的一瞬間＝揮棒**。由「放開時機 vs 球到位時刻」（時機窗依球速決定：直球窄、變化球寬）、放開當下甜蜜點與球的距離、球種三者共同判定結果——時機超窗或位置沒對上都是揮空。球沿真實軌跡飛行：直球直線一閃而逝；滑球前段像直球、中段向外橫移；曲球出手就是大弧線；指叉／變速前段完全像直球、末段才突然下墜。沒出手＝目送（好球帶＝好球；壞球區靠選球眼判定要不要追）
- **打擊模式**：看球（磨球數）／正常打擊／強力打擊（全壘打大增、甜蜜區更小、打偏更傷）／觸擊短打。已取消球種猜測——你賭的是「位置與反應」
- **逐句轉播**：打擊結果以直播感逐句播出（球飛向中外野→外野手跳起→打中全壘打牆→三壘安打）
- **決策回放**：每球結果後揭曉投手球路/落點/佈陣、打者猜法/跑壘企圖與實際落點，方便玩家看懂心理戰勝負手
- **賽後 Box Score**：比賽結束顯示安打、長打、全壘打、奪三振、保送/死球、盜壘、雙殺、投球數與關鍵一擊
- **戰報高亮與房號複製**：得分/全壘打/三振/雙殺等重要事件會上色，等待頁可一鍵複製房號
- **開發 Debug 面板**：本機 `npm run dev` 時結果畫面可展開 Debug，查看本球投打選擇與判定摘要
- **彩蛋：投降輸一半**：連點三下「直球」或「打擊」按鈕可觸發隱藏指令，向對方提出投降；對方的大屏幕會播出經典畫面並選擇是否接受——接受＝發起方認輸、比賽立即結束
- **牽制**：一/二壘有跑者時，投手配球階段可牽制（每打席 2 次）。牽制會重置配球倒數；小機率直接抓到離壘過大的跑者（跑者越慢越危險）、也可能暴傳失誤讓全體跑者推進（失誤率隨疲勞上升，封頂 20%）；跑者平安回壘則「被牽制住」，該球盜壘成功率 -12%/次（可疊加）
- **敬遠**：比照中職自動故意四壞，投手比手勢即保送（不投球、不耗體力）；滿壘敬遠會強迫擠分
- **觸身球**：瞄內角或壞球區時有小機率觸身（控球越差、越疲勞機率越高），死球保送、不可盜壘
- **雙殺佈陣**：第四種佈陣，一壘有人時滾地球雙殺率大增，但安打與長打略增
- **即時異動通知**：換投與代打會立刻以醒目橫幅顯示在雙方畫面

- **投手（等級制）**：每顆球種標示 S/A/B/C——擅長球路依球威分 S/A，陌生球路依綜合能力分 B/C；疲勞每 6 點全等級下修 1 階。S 級位移刁鑽幾乎不失投；C 級失投率高、被打擊率 +10%。落點改為「拖曳白球」到九宮格＋外圈壞球區，變化球即時顯示位移球影（指叉向下墜、滑球外竄）
- **牛棚**：每隊先發/中繼/終結者 3 人，最多換投 2 次，退場不可回。疲勞依投球數累積
- **佈陣**：正常／內野趨前（封鎖三壘跑者）／拉打佈陣（剋強力打擊、被正常推打鑽空檔）／雙殺佈陣
- **盜壘暗號（前置宣告）**：在對方配球階段偷偷下暗號（守備方看不到），投球出手跑者立刻啟動。變化球揮空成功率 +18%、直球 -12%、界外球不算、安打跑者多推進一個壘包；若對方剛好選牽制→大概率直接抓死。一二壘有人只能「雙盜壘」（成功二三壘有人；失敗前位跑者三壘遭觸殺、後位上二壘）；一三壘有人只可盜一壘跑者
- **代打**：板凳 2 人，換上永久取代該棒次
- **雙殺**：一壘有人滾地出局約 24% 成雙殺；跑者已因盜壘暗號啟動時雙殺率驟降至 5%（但揮空跑者幾乎必死）
- **延長賽**（建房時二選一）：「快速決勝」＝突破僵局制（每個延長半局開始，前一棒打者站上二壘）打到分出勝負；「中職例行賽制」＝無突破僵局跑者，延長最多 +3 局，仍平手依例行賽賽制和局。延長局下半主隊超前即再見結束
- 房間閒置 3 小時自動消失

- **9 棒打線＋DH 制**：比照中職，投手不打擊，每隊 9 名打者輪打線＋2 名板凳代打
- **牛棚 4 人全輪替**：先發＋2 中繼＋終結者，換投次數不設上限，改由牛棚人數自然限制（每人只能上場一次），體力血條讓過度用人有實際代價
- **左右打對決**：打者與投手同邊小幅劣勢、異邊小幅優勢（左右開弓恆為異邊），自動計算不需額外操作
- **洋將標記**：畫面上「洋」標籤僅作情報顯示，不限制上場名額，保持操作單純
